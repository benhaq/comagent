# Codebase Summary Template (Portable Baseline)

Use these sections in order unless the target repository has a stronger local convention.

## 1) Repository Layout

- Directory tree showing the top-level structure.

## 2) Key Entry Points

- Paths where execution starts (main, CLI, API routes, workers, etc.).

## 3) Core Modules

- Table: Path | Responsibility.
- Cover the important parts, not every file.

## 4) Data and Integration Paths (Optional)

- Key data flows between modules or external systems.

## 5) Local Development Commands (Optional)

- Setup, test, build, and run commands.

## 6) Related Docs (Optional)

- Links to system-design, specs, rules.
