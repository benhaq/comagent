# Feature Spec Conventions

## Canonical Placement

- Feature spec entry: `docs/specs/{NNNN}-{feature-name}/spec.md`
- Example: `docs/specs/0001-user-authentication/spec.md`

## Folder Convention

- One folder per feature under `docs/specs/`.
- `spec.md` is the required entry file.
- Additional supporting docs are optional.

## Usage

1. Start from `references/feature-spec-template.md`.
2. Populate known context first; leave unknowns in `Open Points`.
3. Keep `spec.md` current as the latest source of truth.
