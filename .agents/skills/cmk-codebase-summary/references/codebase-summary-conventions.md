# Codebase Summary Conventions

## Canonical Placement

- Codebase summary entry: `docs/codebase-summary.md`
- One codebase summary per repository

## Scope Boundary

- Codebase summary owns repository structure and navigation
- Architecture rationale belongs in `docs/system-design.md`
- Feature behavior belongs in `docs/specs/{NNNN}-{feature-name}/spec.md`

## Usage

1. Start from `references/codebase-summary-template.md`.
2. Explore the actual codebase rather than guessing.
3. Keep `docs/codebase-summary.md` current as the repo changes.
